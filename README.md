# Планировщик недели — онлайн-версия

Что это: тот же планировщик, но теперь данные хранятся не в файле на компьютере, а в
облаке (Firebase). Можно завести несколько аккаунтов — у каждого будет свой,
полностью независимый план. Изменения сохраняются автоматически, без кнопки
«Сохранить».

Экспорт/импорт JSON-файла оставлены как ручной бэкап — можно выгрузить копию
своих данных в файл в любой момент (кнопка «Экспорт в файл»).

Всё, что ниже, нужно сделать один раз, чтобы приложение заработало по-настоящему.
Пункты 1–4 делаются в браузере, в консоли Firebase — это твой личный кабинет,
я туда доступа не имею.

## 1. Создать проект в Firebase

1. Открой https://console.firebase.google.com и войди со своим Google-аккаунтом.
2. «Добавить проект» → придумай название (например, `week-planner`) → можно
   отключить Google Analytics (не нужен для этого приложения) → «Создать проект».

## 2. Включить Authentication (вход по email/паролю)

1. В меню слева: **Build → Authentication** → «Начать работу» (Get started).
2. Вкладка **Sign-in method** → выбери **Email/Password** → включи первый
   переключатель («Email/Password») → «Сохранить».

Это всё, что нужно для входа — отдельно создавать пользователей не надо,
регистрация происходит прямо в приложении (кнопка «Зарегистрироваться» на
экране входа).

## 3. Включить Firestore (база данных)

1. **Build → Firestore Database** → «Создать базу данных» (Create database).
2. Выбери ближайший к тебе регион (менять потом нельзя, но для личного
   приложения это не критично).
3. Режим — **Начать в тестовом режиме** (Start in test mode) — не страшно, ниже
   мы заменим правила на собственные (`firestore.rules` уже готов в этой папке).

## 4. Получить конфигурацию веб-приложения

1. Шестерёнка рядом с «Project Overview» → **Project settings**.
2. Внизу раздела «Your apps» → значок `</>` (Add app → Web).
3. Дай приложению любое имя (галочку «Firebase Hosting» можно не ставить —
   мы подключим хостинг отдельно через CLI) → «Register app».
4. Появится блок `const firebaseConfig = { apiKey: "...", ... }` — скопируй его
   целиком.

## 5. Вставить конфигурацию в код

Открой `index.html`, в самом начале `<script type="module">` найди:

```js
const firebaseConfig = {
  apiKey: "ВСТАВЬ_СЮДА",
  authDomain: "ВСТАВЬ_СЮДА.firebaseapp.com",
  projectId: "ВСТАВЬ_СЮДА",
  storageBucket: "ВСТАВЬ_СЮДА.appspot.com",
  messagingSenderId: "ВСТАВЬ_СЮДА",
  appId: "ВСТАВЬ_СЮДА"
};
```

и замени значениями, которые скопировала на шаге 4. `apiKey` для веб-приложений
Firebase не секретный — он просто говорит браузеру, к какому проекту
подключаться; реальная защита данных — в `firestore.rules` (доступ только к
своим документам), это уже настроено.

Также открой `.firebaserc` и замени `ВСТАВЬ_СЮДА_ID_ПРОЕКТА` на **Project ID**
(не название, а именно ID — он виден там же, в Project settings, под названием
проекта).

## 6. Установить Firebase CLI и задеплоить в первый раз

Через терминал (на своём компьютере или попросив меня выполнить это здесь, если
у тебя подключён мост к рабочему столу):

```bash
npm install -g firebase-tools
firebase login                 # откроется браузер для входа в тот же Google-аккаунт
cd week-planner-app
firebase deploy                # зальёт и Hosting, и правила Firestore
```

После этого в терминале появится ссылка вида `https://ТВОЙ-ПРОЕКТ.web.app` —
это и есть постоянный адрес приложения, доступный из интернета.

## 7. Положить код в GitHub (нужно для дальнейшей разработки через Клода)

```bash
git init
git add .
git commit -m "Первая онлайн-версия планировщика"
```

Дальше создай пустой репозиторий на https://github.com/new (без README/лицензии)
и подключи его:

```bash
git remote add origin https://github.com/ТВОЙ-АККАУНТ/week-planner.git
git branch -M main
git push -u origin main
```

## 8. Автодеплой при каждом изменении (необязательно, но удобно)

Файл `.github/workflows/deploy.yml` уже настроен — при каждом пуше в `main`
GitHub сам задеплоит новую версию на Firebase Hosting. Чтобы это заработало,
нужно один раз добавить два секрета в репозиторий (Settings → Secrets and
variables → Actions → New repository secret):

- `FIREBASE_PROJECT_ID` — Project ID из шага 4/5.
- `FIREBASE_SERVICE_ACCOUNT` — JSON-ключ сервисного аккаунта. Получить его
  можно командой `firebase init hosting:github` (она сама создаст ключ и
  предложит добавить его в секреты репозитория за тебя) — это самый простой
  способ, стоит использовать именно его вместо ручного создания ключа в Google
  Cloud Console.

Если пропустить этот пункт — ничего страшного, просто после изменений нужно
будет вручную запускать `firebase deploy`.

## 9. Как продолжать развивать приложение через Клода

Репозиторий на GitHub — это единственный источник правды. В следующий раз
достаточно попросить: «вот ссылка на репозиторий, склонируй и внеси такое-то
изменение» — я внесу правки прямо в `index.html` (или добавлю новые файлы),
закоммичу и запушу в `main`. Если настроен автодеплой (шаг 8) — сайт обновится
сам за минуту-две; если нет — просто скажи, и я подскажу выполнить
`firebase deploy`, либо сделаю это за тебя, если у меня будет доступ к
терминалу с установленным Firebase CLI и твоим логином.

## Структура проекта

- `index.html` — всё приложение (HTML+CSS+JS в одном файле, как и раньше).
- `firebase.json` — настройки хостинга и путь к правилам Firestore.
- `firestore.rules` — правила безопасности (каждый видит только свои данные).
- `firestore.indexes.json` — индексы Firestore (пока не нужны, оставлен пустым).
- `.firebaserc` — какой Firebase-проект использовать по умолчанию.
- `.github/workflows/deploy.yml` — автодеплой на GitHub Actions.
